Как разработчик уровня senior, создай Telegram-бота на Python с использованием aiogram (v3 или выше) и OpenAI API.

#Требования:
1. Бот должен поддерживать память (контекст диалога) — хранить последние N сообщений пользователя и ассистента для каждого чата (например, в словаре или JSON-файле).
2. Подключение к OpenAI API (модель gpt-5 mini).
3. Хранить системные промпты (режимы) в отдельном JSON-файле `prompts.json`, где можно легко добавлять и редактировать режимы.
4. Возможность переключать текущий промпт через команду `/mode` — бот должен показывать список доступных режимов и позволять выбрать.
5. Команда `/reset` очищает память диалога.
6. Код должен быть чистым, структурированным и легко расширяемым.

#Initial setup
Скрипт Telegram-бота должен подключать openai или любой совместимый LLM (gpt-4, gpt-3.5, llama 3 и т.д.), который получен через AI-провайдера, так же как это сделано в текущих скриптах ai_direct.py и openai_client.py с теми же api-ключами как и в файле .env. Не удаляй ключи из текущего файла .env. OPENAI_API_KEY нужен не отдельно, а от того провайдера, которого выбрал пользователь.
#Settings
При первом запуске, пользователь выбирает ai-провайдера, модель, температуру, токены из доступных вариантов в скриптах ai_direct.py и openai_client.py. Пользоватль должен иметь возможность смениь настройки.

#Функциональность:
- При старте бот загружает `prompts.json`.
- У каждого пользователя хранится выбранный режим и история сообщений.
- При получении сообщения бот добавляет его в память, формирует контекст (system_prompt + последние 5 сообщений), отправляет в OpenAI, и возвращает ответ.
- Код должен содержать комментарии и разделы (handlers, utils, config, main).

#Итог:
Сгенерируй полностью рабочий пример проекта, включая файлы:
- `main.py`
- `config.py`
- `memory.py` (или `utils/memory.py`)
- `prompts.json`

Должны быть сгенерироваены файлы config.py с конфигурацией бота, memory.py с управлением памятью диалогов, добавлением сообщений, взаимодействием с файлами и получением истории, и main.py с логикой работы сервера (инициализация, обработчики команд пользователя и запросов Telegram и OpenAI). Логика работы в данном случае асинхронная, для возможности быстрого ответа сразу нескольким пользователям. Кроме того, должны быть файлы prompts.json с нашими заданными промптами ролей и requirements.txt с требуемыми библиотеками. В качестве зависимостей мы задаём aiogram, OpenAI и python-dotenv для загрузки переменных среды.
